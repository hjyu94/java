# 자료형
    - 기본형(primitive): 메모리의 크기가 정해져 있다
    - 참조형: JDK에서 제공되는 클래스나 프로그래머가 정의하는 클래스. 클래스에 따라 메모리의 크기가 다르다
    (String, Student, ...)

# 리터럴
    문자나 숫자

# long ?
    기본적으로 자바에서는 리터럴이 있으면 4바이트 공간에 저장.

    int iVal = 12345678900; (X)
    -> 오버플로우 에러

    long lVal = 12345678900; (X)
    4바이트로 처리할 수 없어서 빨간 줄
    따라서 이 리터럴은 8바이트에 저장하라는 뜻에서 L을 붙인다.
    (long lVal = 12345678900L;)

    long lVal = 100; (O)
    리터럴 100을 4바이트에 저장 후 자동 형변환 해서 에러가 나지 않는다.
