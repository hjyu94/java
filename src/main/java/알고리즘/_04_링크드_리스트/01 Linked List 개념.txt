# Linked List
    자료를 저장하는 구조의 한 종류
    일렬로 데이터를 저장할 때 사용
    '데이터'와, '다음 데이터를 가리키는 주소'를 저장.

cf) 배열
    배열 방들이 물리적으로 한 곳에 모여있다.
    배열의 크기를 한번 정하면 늘리거나 줄일 수 없다.
    이에 반해 링크드 리스트는 데이터를 삽입하거나 삭제할 때
    관련 노드들이 가진 '다음 노드를 가리키는 포인터'를 변경하면 된다.

    주소를 일일히 찾아다녀야 해서 배열과 비교하면 검색에 속도가 느리지만,
    삽입 삭제는 크기가 제한적이지 않은 링크들 리스트를 사용하는게 성능상 좋다.
