# Trie

문자열에서 검색을 빠르게 해주는 자료구조

정수형 자료에 대해 이진트리로 검색하면 O(logn)의 시간이 걸림.
한번 검색할 때마다 검색할 양이 반씩 줄어드니까.

문자열 자료라면 한 노드안에 들어가서 char[0], char[1], ... 하나하나 검색해야 함.
문자열의 최대 길이가 M 이라고 할 때 O(Mlongn) 의 시간복잡도가 걸린다.

이를 해결하기 위해 Trie 라는 트리를 사용한다.

사전구조라고 생각하고 한 노드에 한 글자씩 채운다.
모든 단어가 같은 문자로 시작하지 않으니 루트 노드는 비운다.

알파벳만 사용한다고 할 때 한 레벨에 존재할 수 있는 노드의 갯수가 최대 26개이다.
                            ()
            (c)                         (d)
    (a)            (o)                  (o)
(r)     (t)     (w)     (p)         (g)     (l)
                        (y)                 (l)

car, cat, cow, cop, copy, dog, doll

M이 문자열의 최대 길이라고 할 때 검색에 O(M) 의 시간복잡도가 들어간다.
