# 입출력 스트림이란?

    자바              ----(입력 스트림)--->    입출력
    응용 프로그램      <---(출력 스트림)----     자료

    - 네트웍에서 자료의 흐름이 물과 같다는 의미에서 유래
    - 다양한 입출력 장치에 독립적으로 일관성 있는 입출력 방식 제공
    - 입출력이 구현되는 곳에서 모두 I/O 스트림을 사용 (키보드, 파일 디스크, 메모리)
    - 스트림 없이 읽거나 쓸 수 없다!


# 입출력 스트림 구분?
    - I? O?
        : 입력 스트림, 출력 스트림 (둘 다 할수는 없다)

    - 자료에 따라서
        : 바이트 스트림, 문자 스트림
        (바이트 스트림: 한바이트씩 읽거나 쓴다. 동영상이나 음악 파일)
        (문자 스트림은 워낙 자주 써서 따로 만들어져 있고, 2바이트씩 처리한다)

    - 스트림의 기능에 따라서
        : 기반 스트림, 보조 스트림
        (자바의 I/O는 데코레이터 패턴으로 구현이 되어있다 -> 나중에 같이 공부할 것!)
        (보조 스트림은 사실 읽거나 쓰는 기능은 없다. 기반 스트림을 서포트 해주는 역할을 한다.)
        (ex: 바이트 -> 문자, 버퍼링 기능, 직렬화 기능 등을 기반 스트림에 제공)
        (기반 스트림이 실제 읽거나 쓰는 기능을 수행)


# 입출력 스트림 종류
    - 입력 스트림: FileInputStream, FileReader, BufferedInputStream, BufferedReader, ...
    - 입력 스트림: FileOutputStream, FileWriter, BufferedOutputStream, BufferedWriter, ...

# 바이트 스트림, 문자 스트림
    - Stream이 붙은 것들은 바이트 스트림이다. 한 바이트씩 읽거나 쓴다)
    - (w+)Reader, (w+)Writer는 문자 스트림이다)

# 기반 스트림, 보조 스트림
    - 기반 스트림: FileInputStream, FileReader, ...
    - 보조 스트림: InputStreamReader, BufferedInputStream, ...
    (데코레이터 패턴으로 기능들을 붙였다 떼었다 할 수 있다)
